## 制限について

ここで使う言語・フレームワークについては **自由** である。しかし、途中での言語・フレームワークの変更は禁止とする。変更する際は、変更前のスタックで実装の最低条件を満たしてから変更となる。

> 最低条件には　`require`をつける

## 共通仕様（全コース共通）

### Todoの基本データ構造

| name | detail | type |
| ---- | ------ | ---- |
|  id  | 一意のID | number / str |
| title | タスク名 | str |
| description | 詳細 | str (/ text) |
| startDate | 開始日時 | datetime |
| endDate | 終了日時 | datetime |
| completed | 完了したか | bool |
| createdAt | 作成日時 | datetime |
| updatedAt | 更新日時 | datetime |

> completed 以外にも、 statusでもOK

> datetimeは基本JST

### 基本機能（CRUD） `require`
- Create - 新しいTodoを追加
- Read - Todo一覧を表示 （個別表示も）
- Update - Todoの編集・完了状態の切り替え
- Delete - Todoの削除

### テストを書く

全ての機能・画面に対してテストできるようにする

## ⚙️ バックエンドコース

### 概要
REST APIの設計・実装に集中。フロントは不要（Postman等でテスト）。

### 必須要件：APIエンドポイント `require`

| メソッド | エンドポイント | 機能 |
|----------|---------------|------|
| `GET` | `/api/todos` | Todo一覧取得 |
| `GET` | `/api/todos/:id` | 特定のTodo取得 |
| `POST` | `/api/todos` | 新規Todo作成 |
| `PUT` | `/api/todos/:id` | Todo更新 |
| `DELETE` | `/api/todos/:id` | Todo削除 |

### レスポンス例

```json
// GET /api/todos
{
  "success": true,
  "data": [
    {
      "id": "1",
      "title": "牛乳を買う",
      "completed": false,
      "createdAt": "2025-01-01T00:00:00Z"
    }
  ]
}

// POST /api/todos（リクエストボディ）
{
  "title": "牛乳を買う"
}

// エラーレスポンス
{
  "success": false,
  "error": "Todo not found"
}
```
